# LibCSS Property System Guide

## Overview

LibCSS uses `properties.toml` as the **single source of truth** for all CSS property metadata. A Python-based code generation system produces most boilerplate files automatically. Understanding the TOML schema and what's generated vs manual is essential for adding/modifying properties.

## Architecture

```
properties.toml ──→ property_generator.py ──→ .inc files, dispatch.c, properties.gen
                                          └──→ gen_parser ──→ autogenerated_*.c parsers
                 ──→ select_generator.py  ──→ computed style getters/setters
                 ──→ cmake_helpers.py     ──→ configure-time property lists
```

## Key Files

### 1. properties.toml (Single Source of Truth)
**Path:** `contrib/libcss/properties.toml`

Defines **all** CSS property metadata: enum values, inheritance, parsing specs, cascade types, computed style config, and more.

```toml
[border_top_width]
enum = "CSS_PROP_BORDER_TOP_WIDTH"
inherited = false
wrap = "css__parse_border_side_width"
cascade = "border_width"
computed = "auto"
bits = 3
initial = "MEDIUM"
parse = "WRAP:css__parse_border_side_width"

[fill]
enum = "CSS_PROP_FILL"
inherited = true
cascade = "bg_border_color"
computed = "auto"
bits = 2
initial = "COLOR"
initial_data = ['0x000000ff']
values = "color"
parse = "IDENT:( INHERIT: INITIAL: REVERT: UNSET: NONE:0,FILL_NONE IDENT:) COLOR:FILL_SET"
```

#### TOML Fields Reference

| Field | Required | Description |
|-------|----------|-------------|
| `enum` | Yes | C enum name, e.g. `CSS_PROP_FILL` |
| `inherited` | Yes | Whether property inherits (boolean) |
| `parse` | No | Parsing spec for gen_parser (omit for SHORTHAND/MANUAL) |
| `cascade` | Yes* | Cascade type: `ident`, `length`, `length_auto`, `bg_border_color`, `manual`, `none` |
| `computed` | Yes* | Computed style: `auto` (generated) or `manual` |
| `bits` | Yes* | Bit width in computed style storage |
| `initial` | Yes* | Initial value keyword |
| `initial_data` | No | Extra initial data (e.g. color values) |
| `values` | No | Value type hint for computed style |
| `wrap` | No | Shared parser function name |
| `manual` | No | Set `true` for fully hand-written parsers |
| `shorthand` | No | Set `true` for shorthand properties |
| `opcodes_manual` | No | Set `true` to skip opcode generation |

*Required for properties with `enum` starting with `CSS_PROP_`

#### Generic (WRAP) Entries

Shared parser functions used by multiple properties:

```toml
[generic.border_side_style]
parse = "IDENT:( INHERIT: INITIAL: REVERT: UNSET: NONE:0,BORDER_STYLE_NONE ... IDENT:)"
```

These generate `autogenerated_border_side_style.c` etc.

### 2. propstrings.h / propstrings.c (String Mappings)
**Paths:** `contrib/libcss/src/parse/propstrings.h` and `.c`

Contains keyword enums and SMAP string mappings.

- **Property name section** (auto-generated): `#include "propstrings_enum.inc"` — comes FIRST
- **Keyword section** (manual): tokens, pseudo-classes, CSS values, colors — comes AFTER

**propstrings.c** includes two generated files:
- `propstrings_strings.inc` — SMAP entries for property names
- `keywords_strings.inc` — SMAP entries for keywords (auto-derived from enum names)

#### Range Markers

The following enum names are **range markers** (skipped by the SMAP generator):
`FIRST_PROP`, `LAST_PROP`, `FIRST_COLOUR`, `LAST_COLOUR`,
`FIRST_SYSTEMCOLOUR`, `LAST_SYSTEMCOLOUR`,
`FIRST_DEPRECATEDCOLOUR`, `LAST_DEPRECATEDCOLOUR`, `LAST_KNOWN`

If you add new range markers, update `PROPSTRINGS_RANGE_MARKERS` in `property_generator.py`.

**WARNING:** Names like `FIRST_CHILD`, `LAST_CHILD`, `FIRST_LINE` are REAL CSS identifiers, NOT range markers.

### 5. properties.h (Value Enums)
**Path:** `contrib/libcss/include/libcss/properties.h`

Contains `css_properties_e` (auto-generated via `#include "properties_enum.inc"`) and **manually written** value enums like `css_fill_e`, `css_display_e`, etc.

## Common Tasks

### Add a Simple IDENT Property (e.g., text-wrap)

**Mostly automatic — just TOML + value enum:**

1. **Add TOML entry** in `properties.toml`:
   ```toml
   [text_wrap]
   enum = "CSS_PROP_TEXT_WRAP"
   inherited = true
   cascade = "ident"
   computed = "auto"
   bits = 2
   initial = "WRAP"
   parse = "IDENT:( INHERIT: INITIAL: REVERT: UNSET: WRAP_STRING:0,TEXT_WRAP_WRAP NOWRAP:0,TEXT_WRAP_NOWRAP BALANCE:0,TEXT_WRAP_BALANCE IDENT:)"
   ```

2. **Add value enum** in `properties.h`:
   ```c
   enum css_text_wrap_e {
       CSS_TEXT_WRAP_INHERIT = 0x0,
       CSS_TEXT_WRAP_WRAP = 0x1,
       CSS_TEXT_WRAP_NOWRAP = 0x2,
       CSS_TEXT_WRAP_BALANCE = 0x3
   };
   ```

3. **Add keyword strings** to `propstrings.h` (if not already present):
   ```c
   BALANCE,  /* after existing keywords */
   ```
   (No need to edit `propstrings.c` — keyword SMAP entries are auto-generated)

4. **Build** — everything else is auto-generated.

### Add a Value Keyword to propstrings
1. Add enum entry to **`propstrings.h`** (manual section, after similar keywords)
2. That's it — `keywords_strings.inc` is auto-generated from enum names
3. Use `c->strings[KEYWORD_NAME]` in parser code

### Add a Property with Manual Parser
1. Add TOML entry with `manual = true` and no `parse` field
2. Create `src/parse/properties/property_name.c` with `css__parse_property_name()`
3. Add `.c` file to CMakeLists.txt library sources

### Add a Property with WRAP (shared parser)
1. Add TOML entry with `wrap = "css__parse_shared_function"` and matching `parse = "WRAP:..."`
2. If the shared function doesn't exist, add a `[generic.shared_function]` entry with its parse spec

### Add a Shorthand Property
1. Add TOML entry with `shorthand = true`
2. Create manual parser `.c` file
3. Add `.c` file to CMakeLists.txt library sources

## Multi-Token Values (e.g., "row dense")
Auto-generated IDENT parsers match **single tokens only**. For multi-token values:
- Set `manual = true` in TOML
- Write parser that consumes multiple tokens in sequence

## Regenerating Files
Files are regenerated automatically during the CMake build via `property_generator.py`.
To force regeneration, touch `properties.toml` or `propstrings.h`.

## ⚠️ Unit Conversion Pitfall

LibCSS has **two separate unit systems** that must be kept in sync:

### Bytecode Units (UNIT_*)
**Path:** `contrib/libcss/src/bytecode/bytecode.h`

Bit-packed enums used during parsing and stored in bytecode:
```c
UNIT_LENGTH = (1 << 8),
UNIT_PX = (1 << 8) + 0,
UNIT_EM = (1 << 8) + 1,
UNIT_PCT = (1 << 9),
UNIT_SIZING = (1 << 14),
```

### CSS Units (CSS_UNIT_*)
**Path:** `contrib/libcss/include/libcss/types.h`

Flat sequential enums used in computed styles and exposed to consumers.

### Translation Function
**Path:** `contrib/libcss/src/select/helpers.h`

`css__to_css_unit()` converts bytecode units to CSS units. **When adding new units:**
1. Add `UNIT_X` to `bytecode.h` in the appropriate category
2. Add `CSS_UNIT_X` to `types.h`
3. Add case to `css__to_css_unit()` in `helpers.h`

**Symptom of missing conversion:** Width/height computes to 0 or unexpected value.

## ⚠️ Manual Parser Structure (calc() support)

When writing a **MANUAL** parser that needs to support `calc()`, follow the generated parser structure:

### ✅ Correct Pattern (else-if chain)
```c
token = parserutils_vector_iterate(vector, ctx);

if ((token->type == CSS_TOKEN_IDENT) && /* keyword check */) {
    error = /* handle keyword */;

} else if ((token->type == CSS_TOKEN_FUNCTION) &&
        (lwc_string_caseless_isequal(token->idata, c->strings[CALC], &match) == lwc_error_ok && match)) {
    /* DO NOT reset ctx - css__parse_calc expects ctx at calc token */
    error = css__parse_calc(c, vector, ctx, result, buildOPV(CSS_PROP_X, 0, X_CALC), UNIT_PX);

} else {
    /* Length/percentage fallback */
    *ctx = orig_ctx;
    error = css__parse_unit_specifier(...);
}
```

**Key insight:** `css__parse_calc()` expects `ctx` to be positioned **at** the calc function token, not before it.

## 🔧 Extending the Parser Generator (gen_parser)

**Path:** `contrib/libcss/src/parse/properties/gen_parser/css_property_parser_gen.c`

### Available Directives

| Directive | Purpose | Example |
|-----------|---------|---------|
| `IDENT:( key:val ... IDENT:)` | Keyword matching | `AUTO:0,WIDTH_AUTO` |
| `LENGTH_UNIT:( ... LENGTH_UNIT:)` | Dimension/percentage values | `UNIT_PX:WIDTH_SET` |
| `CALC:( unit:value CALC:)` | calc() function support | `UNIT_PX:WIDTH_CALC` |
| `SIZING:( key:val ... SIZING:)` | Content-sizing keywords | `FIT_CONTENT:WIDTH_SET` |
| `COLOR:value` | Color parsing | `COLOR:COLOR_SET` |
| `NUMBER:( ... NUMBER:)` | Numeric values | `true:Z_INDEX_SET` |
| `URI:value` | URL values | `URI:LIST_STYLE_IMAGE_URI` |
| `WRAP:function` | Delegate to shared function | `WRAP:css__parse_border_side_style` |
